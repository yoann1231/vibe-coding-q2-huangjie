## 作业实施规格书（执行版）

### 1. 目标

- 交付可自动调用的 Claude Skills 体系，完成 Amazon 热品到 Temu/4supply 上架机会再到 1688 供货核价的闭环。
- 严格执行 V4.1 利润铁律，所有候选品必须经过统一核价后再输出推荐。
- 最终输出 5 款产品推荐，且每款均包含前端营销逻辑与后端生存逻辑。

### 2. 强约束规则

- 市场源头必须来自 Amazon Movers & Shakers 近期飙升趋势。
- 候选品必须匹配以下策略之一：Bundle、Premium、Lightweight。
- 回款预估（USD）= Temu 卷王价 × 45%。
- 回款预估（CNY）= 回款预估（USD）× 7.2。
- 总成本（CNY）= 1688 起批价 + 3.5。
- 净利润（CNY）= 回款预估（CNY）- 总成本。
- 净利润必须大于 5.00，否则一律 PASS。

### 3. Skill 拆解

- Skill A：Amazon 趋势发现器，输出飙升品、增长原因与场景标签。
- Skill B：Temu/4supply 空档分析器，判断未卖爆和切入机会等级。
- Skill C：避卷策略生成器，给出 Bundle、Premium、Lightweight 落地方案。
- Skill D：1688 供货助手，输出找货关键词、询盘话术与成本区间。
- Skill E：V4.1 利润闸门，执行统一核价并给出 GO/PASS。
- Skill F：报告生成器，按题目格式产出 5 款推荐结果。

### 4. 复用优先策略

- 先调研官方与市面 Skills 渠道，再决定自研。
- 对每个能力做复用判定：可直接复用、可二次封装、必须自研。
- 关键利润计算能力必须由自研 Skill E 统一输出，避免口径不一致。

### 5. 实施节奏（4 天）

- 第 1 天：完成需求冻结、Skill 拆解、市场调研与复用清单。
- 第 2 天：优先实现 Skill E 与 Skill F，形成可演示最小闭环。
- 第 3 天：实现 Skill A、B、C、D，并完成串联。
- 第 4 天：联调、回归测试、安装命令整理、录屏与交付打包。

### 6. 验收用例

- 用例 1：净利润 4.99 时必须 PASS。
- 用例 2：净利润 5.01 时必须 GO。
- 用例 3：每个候选品都必须输出一个避卷策略。
- 用例 4：最终报告每款都必须含两段逻辑结构。
- 用例 5：缺失价格或成本参数时返回结构化错误。

### 7. 交付清单

- GitHub 分支：skills名称-候选人姓名拼音全称。
- 可一行命令安装的 Skills。
- 每个 Skill 的输入输出样例与测试结果。
- 全链路演示视频（Skill 调用、前后端测试、报告产出）。
